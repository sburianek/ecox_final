★ Worked Problem 2 — the employment model, all ten parts

This is the specimen's actual P2, solved end-to-end with the real appendix critical values. It is the single best rehearsal for exam day: every technique you need appears here once. The estimated model (a linear probability model — the dependent variable is binary):

\[ \widehat{\text{employed}}=0.61+0.051\,\text{age}-0.0005\,\text{age}^2+0.015\,\text{workexp}+0.020\,\text{educ}+0.124\,\text{male}+0.062\,\text{white} \]

standard errors: (3.44) (0.015) (0.0001) (0.0024) (0.039) (0.031) (0.009)
\(n=307\), \(R^2=0.60\), \(\bar R^2=0.592\), \(\text{SSR}=15\). So \(k=6\), df \(=307-6-1=300\) (use the table's ∞ row — the closest available).

Part 1 (3 pts) — interpret β̂(male), significance, 95% CI

Interpretation. Since this is an LPM, the coefficient is a change in a probability, in percentage points. Being male (vs the female base group) is associated with a 12.4 percentage-point higher probability of employment, holding age, experience, education and ethnicity fixed.
Significance (t test). \(t=\dfrac{\hat\beta}{se}=\dfrac{0.124}{0.031}=4.0\). Two-sided 5% critical value (df ≈ ∞) is \(1.96\). Since \(4.0>1.96\), significant at 5% (indeed at 1%, where the crit is 2.576).
95% CI. \(\hat\beta\pm1.96\,se=0.124\pm1.96(0.031)=0.124\pm0.061=[\,0.063,\ 0.185\,]\). Excludes 0, consistent with the test.

Part 2 (3 pts) — does being white raise P(employed) by more than 5 pp? (one-sided, nonzero null)

Key idea. The null value is 0.05, not 0 — you subtract it in the numerator. This is the classic trap.
Hypotheses. \(H_0:\beta_{white}=0.05\) vs \(H_1:\beta_{white}>0.05\) (one-sided).
Statistic. \(t=\dfrac{\hat\beta-0.05}{se}=\dfrac{0.062-0.05}{0.009}=\dfrac{0.012}{0.009}\approx1.33\), distributed \(t_{300}\).
Critical value. One-sided, 10% → use the 0.10 column, df ≈ ∞ → \(c=1.282\).
Conclusion. \(1.33>1.282\) ⇒ reject \(H_0\) at 10%: the white effect is statistically greater than 5 pp — but only just (it would not survive a 5% test, where \(c=1.645\)). Economically, ~6 pp is a modest employment gap.

Part 3 (3 pts) — total effect of age, effect at age 30, turning point

Key idea. Age enters as a quadratic (age and age²), so its effect is not constant — it changes with the level of age. Differentiate.
Marginal effect. \(\dfrac{\partial\widehat{\text{employed}}}{\partial\,\text{age}}=\beta_1+2\beta_2\,\text{age}=0.051+2(-0.0005)\,\text{age}=0.051-0.001\,\text{age}\).
At age 30. \(0.051-0.001(30)=0.021\) → one more year of age raises P(employed) by about 2.1 pp at age 30. Positive but diminishing — as expected, employment rises with age early on.
Turning point. \(\text{age}^*=-\dfrac{\beta_1}{2\beta_2}=-\dfrac{0.051}{2(-0.0005)}=51\) years. Since \(\beta_2<0\) this is a maximum: P(employed) rises until ~51, then falls. 51 is well within the adult age range, so the curvature is practically relevant.

Part 4 (2 pts) — why is education insignificant? a better representation?

Reasoning. Years of education may have a non-linear or threshold effect on employment — what matters is completing a degree, not each marginal year. A linear "years" term averages this away and can look insignificant.
Better representation. Replace the continuous years variable with education-level dummies (e.g. secondary / bachelor / master), capturing credential effects; or add a quadratic in education; or interact it with another variable. Any captures non-linearity the linear term misses.

Part 5 (1 pt) — interpret R²

\(R^2=0.60\): the model explains 60% of the variation in the employment outcome across individuals. The remaining 40% is unexplained (in the residual). A respectable fit for cross-sectional micro-data.

Part 6 (2 pts) — overall significance F (R² form)

Key idea. Tests \(H_0\): all slopes \(=0\) jointly. The hint says use the R² form, which needs only \(R^2\), \(k\), and df.
\(F=\dfrac{R^2/k}{(1-R^2)/(n-k-1)}=\dfrac{0.60/6}{0.40/300}=\dfrac{0.10}{0.001333}=75.0\), distributed \(F_{6,300}\).
Conclusion. The 1% critical value (df ≈ 6, denom ∞) is well under 3; \(75\gg3\) ⇒ reject — the regression is jointly highly significant.

Part 7 (3 pts) — joint test H₀: β₃ = 0 and β₄ = 0 (SSR form)

Key idea. Drop age² and workexp, re-estimate (the "restricted" model), and compare residual fit. Given: \(\text{SSR}_U=15\) (full), \(\text{SSR}_R=16\) (restricted), \(q=2\) restrictions.
Hypotheses. \(H_0:\beta_3=0\ \text{and}\ \beta_4=0\) vs \(H_1:\) at least one \(\neq0\).
Statistic. \(F=\dfrac{(\text{SSR}_R-\text{SSR}_U)/q}{\text{SSR}_U/(n-k-1)}=\dfrac{(16-15)/2}{15/300}=\dfrac{0.5}{0.05}=10.0\), distributed \(F_{2,300}\).
Critical value & conclusion. 1% crit, \(F_{2,120}\approx4.79\) (closest table row); \(10>4.79\) ⇒ reject \(H_0\): the two variables are jointly significant at 1%.

Part 8 (2 pts) — the LPM's known heteroskedasticity & the WLS fix

Key idea. In an LPM the error variance is not a free unknown — it's pinned down by the fitted probability. For a Bernoulli outcome, \(\mathrm{Var}(u\mid X)=p(X)\,[1-p(X)]\), and \(p(X)=\widehat{\text{employed}}=\hat y\).
The form, specifically. \(\mathrm{Var}(u_i\mid X)=\hat y_i(1-\hat y_i)\), where \(\hat y_i\) is each observation's fitted probability.
WLS procedure. Weight each observation by \(1/\sqrt{\hat y_i(1-\hat y_i)}\) — i.e. divide the entire equation (y and every regressor including the constant) by \(\sqrt{\hat y_i(1-\hat y_i)}\), then run OLS on the transformed data. (If any \(\hat y_i\notin(0,1)\), that weight is undefined — drop or adjust those observations.)

Part 9 (3 pts) — RESET test, p-value 0.352

Idea. RESET checks for functional-form misspecification: it adds powers of the fitted values (\(\hat y^2,\hat y^3\)) to the model and jointly tests whether they matter. If they do, the original linear form missed some curvature.
Interpretation here. p = 0.352 is large (> any usual α), so we fail to reject the null of correct specification — no evidence of misspecification. The functional form is adequate.
If it had flagged misspecification (small p), you'd try a more flexible form: add quadratic/interaction terms, or use logs of the continuous regressors.

Part 10 (3 pts) — omitted variables & bias-sign analysis

Key idea. A variable left in the error biases OLS only if it is BOTH a real determinant of employment AND correlated with an included regressor. Sign: bias \(=\operatorname{sign}(\delta)\times\operatorname{sign}(\text{corr})\), where \(\delta\) is the omitted variable's true effect.
Candidate 1 — health status. Better health raises employment (\(\delta>0\)) and is positively correlated with age up to a point / with education → omitting it biases those coefficients upward (same signs).
Candidate 2 — motivation / ability. Raises employment (\(\delta>0\)) and is positively correlated with education → the education coefficient is biased upward; part of what looks like an "education effect" is really motivation.
State, for each: its effect sign, its correlation sign with a named regressor, and therefore the bias direction. That structure is what earns the marks.

1 · Population model & OLS estimators

The model, and the words for its pieces P1 · P4

The population model says the truth about how y is generated:

\[ y=\beta_0+\beta_1x_1+\dots+\beta_kx_k+u,\qquad E(u)=0. \]
Definitions — the four object types Parameter \(\beta_j\): the true, fixed, unknown number. · Estimator \(\hat\beta_j\): the formula that turns a sample into a guess — a random variable, since it depends on which sample you drew. · Estimate: the actual number that formula spits out for your data. · Error \(u\): the unobservable population disturbance, \(y-E(y\mid x)\). · Residual \(\hat u_i=y_i-\hat y_i\): its observable sample counterpart.
Population (true, unknown, hatless)Sample (estimated, hatted)
parameter \(\beta_j\)estimate \(\hat\beta_j\)
expected value \(E(y\mid x)\)fitted value \(\hat y_i\)
PRF (population regression function) \(E(y\mid x)=\beta_0+\beta_1x_1+\dots\)SRF (sample regression function / OLS line) \(\hat y=\hat\beta_0+\hat\beta_1x_1+\dots\)
error \(u\)residual \(\hat u\)

The PRF is fixed but unknown; the SRF is your estimate of it and changes from sample to sample. That sample-to-sample variability is precisely what gives \(\hat\beta_j\) a sampling distribution (and hence a standard error).

The OLS estimators — what they are P4

OLS = Ordinary Least Squares: the line that minimises the sum of squared residuals. For simple regression (one x):

\[ \hat\beta_1=\frac{\sum_{i=1}^n (x_i-\bar x)(y_i-\bar y)}{\sum_{i=1}^n (x_i-\bar x)^2},\qquad \hat\beta_0=\bar y-\hat\beta_1\bar x. \]

Read \(\hat\beta_1\) as "sample covariance of x and y, divided by sample variance of x." For multiple regression, stack everything into matrices (X = data, y = outcomes):

\[ \hat{\boldsymbol\beta}=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y. \]

Sums of squares — the building blocks of fit P4

Definitions SST (total) \(=\sum(y_i-\bar y)^2\): total variation in y to be explained. · SSE (explained) \(=\sum(\hat y_i-\bar y)^2\): variation the model captures. · SSR (residual) \(=\sum\hat u_i^2\): variation left over. They satisfy \(\text{SST}=\text{SSE}+\text{SSR}\).
Notation trapSome books swap the names SSE/SSR. Here SSE = Explained, SSR = Residual (\(\sum\hat u^2\)) — Wooldridge's convention. Use it consistently.

Interpreting coefficients P2

  • Intercept \(\hat\beta_0\): predicted y when every \(x_j=0\) (often not meaningful on its own, but needed for the line).
  • Slope \(\hat\beta_j=\dfrac{\Delta\hat y}{\Delta x_j}\): the partial effect — the change in predicted y for a one-unit rise in \(x_j\), holding all other regressors fixed.

Logarithmic specifications (Problem 4, item 10 — and everywhere in P2)

Why logs? They convert absolute changes into percentage changes, giving elasticities economists care about.

ModelDep.Indep.Meaning of \(\beta_1\)
Level–level\(y\)\(x\)\(\Delta y=\beta_1\Delta x\): units of y per unit of x
Level–log\(y\)\(\log x\)\(\Delta y=(\beta_1/100)\%\Delta x\): units of y per 1% rise in x
Log–level\(\log y\)\(x\)\(\%\Delta y=(100\beta_1)\Delta x\): % change in y per unit of x (semi-elasticity)
Log–log\(\log y\)\(\log x\)\(\%\Delta y=\beta_1\%\Delta x\): elasticity
Memory ruleLog on the dependent side → ×100 to read y's % change. Log on the independent side → ÷100. Both logged → \(\beta_1\) is the elasticity.
Precision note: \(100\beta_1\) is an approximation (good for small \(\beta_1\)); exact %Δy from a one-unit Δx in log-level is \(100(e^{\beta_1}-1)\). For \(\beta_1=0.5\): 50% approx vs 64.9% exact — flag this for marks.

2 · Unbiasedness, consistency & variance

The MLR assumptions, and what each one buys P1 · P3

Definitions of the key properties Unbiased: \(E(\hat\beta_j)=\beta_j\) — right on average across samples. · Consistent: \(\hat\beta_j\to\beta_j\) as \(n\to\infty\). · Efficient / BLUE: smallest variance among all linear unbiased estimators. · BUE: smallest variance among all unbiased estimators (a stronger claim).
#Assumption (plain words)Buys you
MLR.1Model is linear in the parametersMLR.1–4 ⇒ unbiased & consistent
MLR.2Data are a random sample
MLR.3No perfect collinearity (no regressor is an exact linear combo of others; X has full rank)
MLR.4Zero conditional mean \(E(u\mid x)=0\): the error is unrelated to the regressors on average
MLR.5Homoskedasticity: \(\mathrm{Var}(u\mid x)=\sigma^2\), constant for all x+MLR.5 ⇒ BLUE (Gauss–Markov)
MLR.6Normality: \(u\sim N(0,\sigma^2)\)+MLR.6 ⇒ BUE & exact small-sample t, F
The ladder — recite it1–4 → unbiased/consistent · +5 → BLUE · +6 → BUE & exact t/F. MLR.4 is broken by omitted variables and wrong functional form (→ bias). MLR.5 is broken by heteroskedasticity (→ inference invalid, but no bias).
Consistency vs unbiasedness (subtle, examinable)For mere consistency, a weaker MLR.4′ suffices: zero mean AND zero correlation \(\mathrm{Cov}(x,u)=0\). Full mean-independence \(E(u\mid x)=0\) is what you need for unbiasedness.

Variance of OLS — and what makes it big P1 · P4

Why we care: the variance of \(\hat\beta_j\) controls how precise your estimate is — and its square root is the standard error that drives every t-test.

\[ \mathrm{Var}(\hat\beta_j)=\frac{\sigma^2}{\text{SST}_j\,(1-R_j^2)}. \]
What each symbol means \(\sigma^2\) = variance of the error (noise in y). · \(\text{SST}_j=\sum(x_{ij}-\bar x_j)^2\) = how much the regressor \(x_j\) varies in your sample. · \(R_j^2\) = the R² from regressing \(x_j\) on all other regressors — a measure of how collinear \(x_j\) is with them.
Var(β̂ⱼ) goes UP when……and DOWN when…
more noise: \(\sigma^2\) ↑more data: \(n\) ↑
\(x_j\) barely varies: \(\text{SST}_j\) ↓\(x_j\) varies a lot
multicollinearity: \(R_j^2\to1\) (VIF \(=\frac{1}{1-R_j^2}\) ↑)regressors nearly uncorrelated
P1 MCQ (specimen Q4)"What makes Var(β̂ⱼ) increase?" → increased error variance ✓, decreased sample variation in \(x_j\) ✓. Additional observations decrease it ✗.

The standard error — and why it's given, not computed

\[ \hat\sigma^2=\frac{\text{SSR}}{n-k-1},\qquad se(\hat\beta_j)=\frac{\hat\sigma}{\sqrt{\text{SST}_j(1-R_j^2)}}. \]

\(\hat\sigma\) is the standard error of the regression (typical residual size). The \(n-k-1\) divisor makes \(\hat\sigma^2\) unbiased. To get \(se(\hat\beta_j)\) by hand you'd need SSR and the auxiliary \(R_j^2\) — impossible without software, so the exam gives you se (the parenthesised number). Your job: \(t=\hat\beta_j/se\), or back out a missing piece via \(se=\hat\beta_j/t\).

Gauss–Markov & asymptotic normality P1 · P3

Gauss–Markov theorem: under MLR.1–5, OLS is BLUE. Add MLR.6 and it's BUE. Asymptotic normality: even without MLR.6, for large n the standardised estimator \(\dfrac{\hat\beta_j-\beta_j}{se(\hat\beta_j)}\) is approximately \(N(0,1)\) — which is why \(t\) and the normal coincide once df > 120.

3 · Hypothesis testing

The engine of Problem 2. The worked P2 above shows these in action; here's the why behind each.

The t test — single coefficient P2

What it doesAsks whether a coefficient is distinguishable from a hypothesised value \(a_j\) (usually 0) given sampling noise. The "t ratio" \(\hat\beta_j/se\) is the special case \(a_j=0\).
\[ t=\frac{\hat\beta_j-a_j}{se(\hat\beta_j)}\ \sim\ t_{\,n-k-1}. \]
If \(H_1\) is…use the … table columnreject \(H_0\) when
\(\beta_j\neq a_j\) (two-sided)\(\alpha/2\) (split the α between tails)\(|t|>c\)
\(\beta_j>a_j\)\(\alpha\) (right tail)\(t>c\)
\(\beta_j\(\alpha\) (left tail)\(t<-c\)
Most common point-lossUsing the \(\alpha\) column for a two-sided test (or \(\alpha/2\) for one-sided). Two-sided 5% → 0.025 column → 1.96. One-sided 5% → 0.05 column → 1.645.

Statistical vs economic significance P2

Two different questions — answer both Statistical significance = is \(\hat\beta\) large relative to its se? (the t-test; heavily driven by sample size — huge n can make a trivial effect "significant"). Economic significance = is the magnitude large in real-world units? Plug a realistic \(\Delta x\), read off the implied \(\Delta y\), and judge whether anyone would care. A coefficient can have either property without the other — the seminar's ros example was economically negligible (1.2% for a 50-point move) even setting significance aside.

Confidence interval P2

\[ \hat\beta_j\ \pm\ t_{\,n-k-1,\,1-\alpha/2}\cdot se(\hat\beta_j). \]

The range of values for \(\beta_j\) consistent with the data at confidence \(1-\alpha\). Shortcut: a two-sided test rejects exactly when the null value falls outside the CI. For df > 120, the "2-sigma rule" (±1.96 ≈ ±2) gives a quick 5% read.

F test — joint hypotheses P2 · P3 · P4

What it doesTests several restrictions at once (e.g. "these two coefficients are both zero"). Compares a restricted model (restrictions imposed) with the unrestricted one — if imposing them barely raises SSR, the restrictions are plausible.
\[ F=\frac{(\text{SSR}_R-\text{SSR}_U)/q}{\text{SSR}_U/(n-k-1)}=\frac{(R_U^2-R_R^2)/q}{(1-R_U^2)/(n-k-1)}\ \sim\ F_{\,q,\;n-k-1}, \]

\(q\) = number of restrictions, R = restricted, U = unrestricted. The two forms are algebraically identical — pick whichever the data give you (SSRs or R²s). Worked twice in P2 above (overall F = 75 via R²; joint F = 10 via SSR).

LM test (Lagrange multiplier) P3

What it doesAn alternative to the F test for exclusion restrictions, using only the restricted model. Intuition: if the excluded variables truly don't matter, the restricted residuals shouldn't be predictable from them.
  1. Same \(H_0,H_1\) as the F test.
  2. Estimate the restricted model; save residuals \(\tilde u\).
  3. Auxiliary regression: regress \(\tilde u\) on all regressors; get \(R^2_{\tilde u}\).
  4. \(LM=n\,R^2_{\tilde u}\ \overset{a}{\sim}\ \chi^2_q\). Reject if \(LM>c\).
Distribution trap (specimen P1 Q1)With MLR.1–4 holding but 5–6 violated, LM is asymptotically χ² — not exact, not F.
"Fail to reject" ≠ "accept"Not rejecting \(H_0\) = insufficient evidence against it, NOT proof it's true (the effect may be real but imprecisely estimated). Always write "fail to reject" / "not significant."

4 · Goodness of fit

Definitions\(R^2\) = share of the variation in y explained by the model. \(\bar R^2\) (adjusted) = the same, but penalised for the number of regressors, so adding junk variables can lower it.
\[ R^2=\frac{\text{SSE}}{\text{SST}}=1-\frac{\text{SSR}}{\text{SST}},\qquad \bar R^2=1-\frac{\text{SSR}/(n-k-1)}{\text{SST}/(n-1)}. \]

\(R^2\) facts P1

  • In \([0,1]\).
  • Never decreases when a regressor is added.
  • Comparable only across models with the same number of regressors.

\(\bar R^2\) facts

  • Can rise or fall; can go negative.
  • Use it to compare specifications (log vs quadratic) and to catch over-controlling.
  • Asymptotically equals \(R^2\).
MCQ traps (specimen Q2)"\(\bar R^2\) always decreases with an added variable" — false. "always > 0" — false (can be negative). "based on the number of regressors" — true. "always lower than \(R^2\)" — true.

5 · Selecting explanatory variables P3 topic 5

Four criteria for "does this variable belong?" — really the bias–variance trade-off in disguise.

  1. Theory. Is inclusion theoretically justified? Purpose matters: prediction vs testing a specific relationship.
  2. OVB reduction. Do other coefficients shift a lot when this variable is added? Big shifts mean it was confounding them — keep it to avoid omitted-variable bias. (The CEO-salary table: the sales elasticity moved 0.224 → 0.158 → 0.188 as controls entered.)
  3. Adjusted \(\bar R^2\). Does fit improve enough to justify the added parameter?
  4. t and F tests. Significant in the expected direction? F handles dropping several variables at once.
More variables is NOT always betterBad controls (variables on the causal path between x and y) bias your estimate if included. Irrelevant variables don't bias OLS but inflate \(\mathrm{Var}(\hat\beta_j)\) — you lose precision. Include only genuine determinants of y that are plausibly correlated with your regressor of interest.
OVB sign rule (used in P2 Part 10)bias \(=\operatorname{sign}(\delta)\times\operatorname{sign}(\mathrm{corr}(x_{\text{omit}},x_{\text{incl}}))\); same signs → upward, opposite → downward. Formally \(\operatorname{plim}\hat\beta_1=\beta_1+\frac{\mathrm{Cov}(x,u)}{\mathrm{Var}(x)}\), which doesn't vanish as \(n\to\infty\) ⇒ inconsistent.

6 · Predictions P4 topic 11

Predicting \(\hat y_{n+1}\) just means plugging new \(x\) values into the SRF. The exam point is which uncertainty you report.

IntervalCapturesWidth
CI for the mean prediction \(E(y\mid x_0)\)only sampling variance of \(\hat\beta\)narrower
Prediction interval for a specific unit \(y_0\)sampling variance of \(\hat\beta\) plus the individual error variance \(\sigma^2\)wider
Why the prediction interval is widerEven if you knew the true line exactly, an individual firm/person still scatters around it by its own \(u\). Predicting one unit adds \(\sigma^2\) on top of the uncertainty in \(\hat\beta\). The error structure: \(\mathrm{Var}(\hat e_0)=\mathrm{Var}(\hat y_0)+\sigma^2\) — the extra \(\sigma^2\) is the whole difference.

7 · Qualitative variables, dummies & the LPM

Dummy variables P2 · P3 topic 6

DefinitionA dummy is a 0/1 indicator for a category (e.g. male = 1). Technically it behaves like any regressor.
  • Base group = the omitted category, absorbed into the intercept. Each dummy's coefficient is the gap vs that base.
  • Dummies shift the intercept; with interaction terms, they also change slopes.
  • Dummy-variable trap: never include a dummy for every category plus an intercept — that's perfect collinearity (violates MLR.3). Always drop one category.

Chow test P3 topic 6

What it doesTests whether the whole set of coefficients is the same across two groups (e.g. men vs women) — i.e. parameter stability. It's an F test comparing a pooled (restricted) regression against separate group regressions (unrestricted).
\[ F=\frac{[\text{SSR}_{pooled}-(\text{SSR}_1+\text{SSR}_2)]/(k+1)}{(\text{SSR}_1+\text{SSR}_2)/(n-2(k+1))}\ \sim\ F_{\,k+1,\ n-2(k+1)}. \]

\(\text{SSR}_1,\text{SSR}_2\) are from each group separately; \(k+1\) restrictions (all coefficients incl. intercept). Reject ⇒ the groups have different coefficient structures. (Equivalently, run one regression with a full set of group-dummy interactions and F-test them jointly.)

Linear probability model (LPM) P3 topic 7

DefinitionAn LPM is just OLS with a binary dependent variable \(y\in\{0,1\}\). Because \(y\) is Bernoulli, its conditional mean is a probability:
\[ E(y\mid X)=P(y=1\mid X)=\beta_0+\beta_1x_1+\dots+\beta_kx_k. \]

So a coefficient is a change in that probability: \(\Delta P(y=1)=\beta_j\,\Delta x_j\) — read as percentage points (as in the employment model above).

Four shortcomings — memorise (favourite P3 entry) (1) fitted probabilities can fall outside [0,1]; (2) the marginal effect of \(\Delta x_j\) is constant, often unrealistic; (3) the error is inherently heteroskedastic; (4) the error is not normal. (3)+(4) ⇒ usual t/F invalid — yet OLS stays unbiased & consistent under MLR.1–4. Advantages: simple, coefficients directly interpretable as probability effects.
The known-variance fix (P2 Part 8)LPM variance is known: \(\mathrm{Var}(u\mid x)=p(x)[1-p(x)]=\hat y(1-\hat y)\). So WLS weights are \(1/\sqrt{\hat y(1-\hat y)}\) — divide the equation through and run OLS.

8 · Heteroskedasticity high-yield P3 family

DefinitionHomoskedasticity (MLR.5): the error has constant variance for all x. Heteroskedasticity = that fails — the spread of \(u\) depends on x.

Consequences for OLS P1 · P3 topic 8

  1. OLS stays unbiased & consistent (MLR.1–4 untouched): the coefficients, \(R^2\), \(\bar R^2\) are all fine.
  2. The true variance of \(\hat\beta\) rises ⇒ OLS is no longer BLUE, not even asymptotically efficient.
  3. The estimated \(\mathrm{Var}(\hat\beta_j)\) is biased (usually downward) ⇒ reported se too small ⇒ t, CI, F, LM all invalid — even in large samples.
One-line takeawayHeteroskedasticity doesn't bias the coefficients — it breaks the standard errors. Fix the inference, not the point estimates.

Remedy 1 — White robust standard errors P3 topic 8

Recompute se to be valid under heteroskedasticity of any unknown form. Caveat: only asymptotically valid (large n), and they say nothing about the bias/consistency/efficiency of \(\hat\beta\) itself — they just repair the standard errors.

Remedy 2 — tests for heteroskedasticity P3 topic 9

Shared ideaIf the error variance depends on the regressors, then the squared residuals \(\hat u^2\) should be predictable from them. So regress \(\hat u^2\) on regressors and test joint significance.
\[ \textbf{Breusch–Pagan: } \hat u^2=\delta_0+\delta_1x_1+\dots+\delta_kx_k+v. \]
\[ \textbf{White: } \hat u^2=\delta_0+\delta_1x_1+\delta_2x_2+\delta_3x_1^2+\delta_4x_2^2+\delta_5x_1x_2+v\quad\text{or}\quad \hat u^2=\delta_0+\delta_1\hat y+\delta_2\hat y^2+v. \]

Test statistic \(n\,R^2\sim\chi^2\) (or an F). \(H_0\) = homoskedasticity; reject ⇒ heteroskedasticity present. White adds squares and cross-products, so it catches more general forms than BP.

Remedy 3 — WLS P3 topic 10

DefinitionWeighted Least Squares: if the variance form \(h(x)\) in \(\mathrm{Var}(u\mid X)=\sigma^2 h(X)\) is known, weight each observation by \(1/\sqrt{h(x)}\) (divide the whole equation through), then run OLS. This re-equalises the error variance.

If the form is correctly specified, WLS is BLUE and gives valid t/F under MLR.1–4 (no MLR.5 needed). The LPM is the textbook known-form case (\(h=\hat y(1-\hat y)\)).

Remedy 4 — FGLS ★ specimen P3 — highest probability

DefinitionFeasible Generalised Least Squares: WLS for when the variance form is unknown — you estimate it from the data first, then weight. "Feasible" = the weights are estimated rather than known.
Full four-part answer (this was the specimen's P3 verbatim) 1 · When? Heteroskedasticity is present but its form is unknown, so WLS weights aren't directly available.
2 · Steps: (a) run OLS, get residuals \(\hat u\); (b) model the variance — regress \(\log(\hat u^2)\) on the regressors (or their functions), get fitted \(\hat g\); (c) form weights \(\hat h=\exp(\hat g)\); (d) run WLS with weights \(1/\hat h\).
3 · Vs OLS under MLR.1–4: both unbiased & consistent; FGLS is not unbiased in finite samples (weights are estimated) but is consistent and asymptotically more efficient.
4 · Does n matter? Yes — FGLS's good properties are asymptotic, so it relies on large n; in small samples it can underperform OLS.

9 · Functional form & data issues

DefinitionFunctional-form misspecification: you chose the right variables but the wrong shape (e.g. omitted a needed \(x^2\) or a log). The missing term lands in the error and correlates with x ⇒ MLR.4 violated ⇒ OLS biased & inconsistent.

RESET test P3 topic 12 · used in P2 Part 9

\[ \textbf{Ramsey RESET: } y=\beta_0+\beta_1x_1+\dots+\beta_kx_k+\delta_1\hat y^2+\delta_2\hat y^3+\varepsilon. \]

Add powers of the fitted values and jointly test \(\delta_1=\delta_2=0\) (F test). Significant ⇒ misspecification; large p-value (e.g. the specimen's 0.352) ⇒ no evidence of it. A general test — doesn't name the better alternative.

Davidson–MacKinnon test P3 topic 12

\[ y=\beta_0+\beta_1x_1+\dots+\beta_kx_k+\delta_1\hat{\hat y}+\varepsilon. \]

For choosing between two non-nested models (e.g. level vs log): add the rival model's fitted values \(\hat{\hat y}\) as a regressor and t-test them. Significant ⇒ the current model is rejected in favour of the rival. Run it both directions.

Measurement error & proxies P3 topic 13

DefinitionsProxy variable: a stand-in for an unobserved regressor (e.g. IQ for ability) to reduce OVB; a lagged dependent variable can serve as one. CEV (classical errors-in-variables): the measurement error is uncorrelated with the true value, \(\mathrm{Cov}(x^*,e)=0\).
  • Measurement error in y (classical): OLS stays unbiased & consistent — just larger standard errors.
  • Measurement error in x (CEV): causes attenuation bias — \(\hat\beta\) is biased toward zero, and OLS is inconsistent. The bigger the measurement noise, the stronger the pull toward zero.

Also under data issues (MLR.2): missing data, nonrandom samples, and outliers can all distort estimates; outliers especially can swing OLS because it squares residuals.

Problem 4 — the full 11-derivation list

The specimen used items 2/3/8 (linear estimator, sums of squares, \(\sum y=\sum\hat y\)) — your high-yield trio. Prepare those cold; skim the rest.

★ 1 · Deriving OLS — Method of Moments & Least Squares

LS: minimise \(\text{SSR}=\sum(y_i-\hat\beta_0-\hat\beta_1x_i)^2\); the two first-order conditions give \(\sum\hat u_i=0\) and \(\sum x_i\hat u_i=0\). MM: impose the sample versions of \(E(u)=0\) and \(\mathrm{Cov}(x,u)=0\) — the same two normal equations. Both yield identical \(\hat\beta\).

★ 2 · Algebraic properties of OLS

From the two normal equations: \(\sum\hat u_i=0\); \(\sum x_i\hat u_i=0\); \(\sum\hat y_i\hat u_i=0\) (since \(\hat y\) is a linear combo of x's); \(\bar y=\overline{\hat y}\); and \((\bar x,\bar y)\) lies on the line. All follow because OLS is defined to force the first two.

★ 3 · Sums of squares: SST = SSE + SSR

Write \(y_i-\bar y=(\hat y_i-\bar y)+\hat u_i\), square and sum: \(\text{SST}=\text{SSE}+\text{SSR}+2\sum\hat u_i(\hat y_i-\bar y)\). The cross term vanishes because \(\sum\hat u_i=0\) and \(\sum\hat u_i\hat y_i=0\). ∴ \(\text{SST}=\text{SSE}+\text{SSR}\).

4 · OLS in matrix form

Minimise \((y-X\beta)^{\mathsf T}(y-X\beta)\); FOC \(X^{\mathsf T}(y-X\hat\beta)=0\) ⇒ normal equations \(X^{\mathsf T}X\hat\beta=X^{\mathsf T}y\) ⇒ \(\hat\beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y\) (invertible by MLR.3, full rank).

★ 5 · Unbiasedness of OLS

Sub \(y_i=\beta_0+\beta_1x_i+u_i\) into \(\hat\beta_1=\sum w_i y_i\) (weights \(w_i=\frac{x_i-\bar x}{\sum(x_j-\bar x)^2}\)): \(\hat\beta_1=\beta_1+\sum w_i u_i\). Take expectations; \(E(u_i\mid x)=0\) (MLR.4) ⇒ \(E(\hat\beta_1)=\beta_1\). Matrix version: \(\hat\beta=\beta+(X^{\mathsf T}X)^{-1}X^{\mathsf T}u\), \(E(\hat\beta\mid X)=\beta\).

6 · Variance of OLS (incl. misspecified models)

\(\mathrm{Var}(\hat\beta_1)=\frac{\sigma^2}{\text{SST}_x}\) (SLR); \(\frac{\sigma^2}{\text{SST}_j(1-R_j^2)}\) (MLR). In a misspecified (under-specified) model, dropping a relevant regressor lowers the variance of the remaining estimator but introduces bias — the bias–variance trade-off in formula form.

7 · Bias in misspecified models / OVB (summation notation)

Omit a relevant \(x_2\) from \(y=\beta_0+\beta_1x_1+\beta_2x_2+u\). The short regression's slope has \(E(\tilde\beta_1)=\beta_1+\beta_2\,\tilde\delta_1\), where \(\tilde\delta_1\) is the slope of \(x_2\) regressed on \(x_1\). Bias \(=\beta_2\tilde\delta_1\); its sign is \(\operatorname{sign}(\beta_2)\times\operatorname{sign}(\mathrm{corr}(x_1,x_2))\).

★ 8 · Linear estimator

\(\hat\beta_1=\sum w_i y_i\) with non-random \(w_i=\frac{x_i-\bar x}{\sum(x_j-\bar x)^2}\) ⇒ linear in y. Properties \(\sum w_i=0\), \(\sum w_i x_i=1\) (used in the unbiasedness and BLUE proofs).

9 · F test: SSR vs R² form

Start from \(F=\frac{(\text{SSR}_R-\text{SSR}_U)/q}{\text{SSR}_U/(n-k-1)}\); substitute \(\text{SSR}=\text{SST}(1-R^2)\) and cancel SST ⇒ \(F=\frac{(R_U^2-R_R^2)/q}{(1-R_U^2)/(n-k-1)}\). The two are identical.

10 · Effects of scaling & logs

Multiply x by \(c\) ⇒ \(\hat\beta\) divides by \(c\) (t-stat, fit unchanged). Multiply y by \(c\) ⇒ all coefficients multiply by \(c\). Logs change the interpretation per the four-model table (§1).

11 · Predictions: mean vs specific-unit

Mean prediction CI uses only \(\mathrm{Var}(\hat y_0)\); the prediction interval for a single unit adds \(\sigma^2\): \(\mathrm{Var}(\hat e_0)=\mathrm{Var}(\hat y_0)+\sigma^2\), hence wider (§6).

Exam-room protocol

Every hypothesis test — five steps (the marks live here)
  1. State \(H_0\) and \(H_1\) fully — include the null value and the right inequality.
  2. Write the test-statistic formula, then plug numbers.
  3. State the distribution + df (\(t_{n-k-1}\), \(F_{q,n-k-1}\), \(\chi^2_q\)).
  4. State the critical value from the table — which tail/column, and the value.
  5. Decision + plain-language conclusion; add an economic-significance remark where relevant.
The specimen instructions demand exactly this — state the null and alternative completely, the theoretical formula, its distribution, the critical value formally, and a clear conclusion.
  • Compute df first (\(n-k-1\)) — it selects your table row.
  • Match tail to \(H_1\): two-sided → \(\alpha/2\) column; one-sided → \(\alpha\) column. The #1 avoidable error.
  • Subtract the null value when it isn't 0 (the "white > 5pp" case).
  • 2-sigma rule: \(|t|>2\) ≈ significant at 5% for df > 120 — sanity check, but cite 1.96.
  • LPM coefficients are percentage points, not percent.
  • Show all arithmetic — no calculator means they expect every step; partial credit lives there.
  • Never write "accept \(H_0\)" — only "fail to reject."
  • Answer only what's asked, in English, legibly, on the question sheet.
